## 😰 Prop Drilling - Problema mare

### Imaginează-ți

**Un restaurant unde comenzile trec prin 5 ospătari înainte să ajungă la bucătar**

Complicat, lent, confuz!

### Ce este Prop Drilling?

**Trebuie să trimiți date prin App.js → Products → ProductList → ProductItem**

```javascript
// App.js
function App() {
  const [cart, setCart] = useState([]);
  
  return <Products cart={cart} setCart={setCart} />;
}

// Products.js
function Products({ cart, setCart }) {
  return <ProductList cart={cart} setCart={setCart} />;
}

// ProductList.js
function ProductList({ cart, setCart }) {
  return <ProductItem cart={cart} setCart={setCart} />;
}

// ProductItem.js
function ProductItem({ cart, setCart }) {
  // În SFÂRȘIT folosesc datele aici!
  return <button onClick={() => setCart([...cart, item])}>Add</button>;
}
```

**Problema:**
- Products și ProductList NU folosesc cart/setCart
- Doar le transmit mai departe
- Cod repetitiv, greu de întreținut
- Dacă adaugi un nou prop, trebuie să-l adaugi în 4 locuri! 😱

**Ce îngrozitor!** Datele trec prin componente care nici măcar nu le folosesc!

---

## 🎯 Context API - Soluția inteligentă

### Analogie

**Context = Wi-Fi pentru date**

În loc să tragi cabluri (props) prin fiecare cameră (componentă), creezi un router Wi-Fi central și fiecare dispozitiv se conectează direct!

### Cum funcționează?

**3 pași simpli:**

1. **📦 Creăm Context** - Un container special care ține informația
2. **🎁 Împachetăm în Provider** - Plasăm componentele în Context Provider
3. **⚡ Accesăm cu useContext** - Orice componentă folosește hook-ul useContext

## 🔧 Cum funcționează Context?

### Pasul 1: Creăm Context

```javascript
// vote-context.js
import { createContext } from "react";

export const VoteContext = createContext();
```

**Ce face `createContext()`?**
- Creează un "container" special pentru date
- E ca o cutie goală care va ține informații

**Cine apelează `createContext()`?**
- **TU** o apelez manual când importezi React: `createContext()`
- Se execută o singură dată când fișierul se încarcă

**Exemplu vizual:**
```
createContext() 
    ↓
Creează VoteContext (cutia goală)
    ↓
Export pentru a fi folosit în alte fișiere
```

### Pasul 2: Creăm Provider

```javascript
// vote-provider.jsx
import { useState } from "react";
import { VoteContext } from "./vote-context";

export function VoteProvider({ children }) {
  const [votes, setVotes] = useState(0);

  const upVote = () => {
    setVotes(votes + 1);
  };

  const downVote = () => {
    setVotes(votes - 1);
  };

  return (
    <VoteContext.Provider value={{ votes, upVote, downVote }}>
      {children}
    </VoteContext.Provider>
  );
}
```

**Explicație pas cu pas:**

#### 1. Parametrul `children`

```javascript
export function VoteProvider({ children }) {
  // children = componentele înfășurate în Provider
}
```

**Ce este `children`?**
- Toate componentele puse între `<VoteProvider>` și `</VoteProvider>`
- E un prop special în React

**Exemplu:**
```jsx
<VoteProvider>
  <App />           ← Asta e children
  <Header />        ← Și asta
  <Main />          ← Și asta
</VoteProvider>

// children = [<App />, <Header />, <Main />]
```

**Cine trimite `children`?**
- **TU** când folosești `<VoteProvider>` în cod
- React îl transformă automat în prop

#### 2. State și funcții

```javascript
const [votes, setVotes] = useState(0);

const upVote = () => {
  setVotes(votes + 1);
};

const downVote = () => {
  setVotes(votes - 1);
};
```

**Cine apelează aceste funcții?**
- `useState(0)` - **React** o apelează automat când componenta se montează
- `upVote()` - **TU** o apelez manual când user-ul dă click
- `downVote()` - **TU** o apelez manual când user-ul dă click
- `setVotes()` - **upVote/downVote** o apelează când sunt executate

#### 3. Provider cu value

```javascript
return (
  <VoteContext.Provider value={{ votes, upVote, downVote }}>
    {children}
  </VoteContext.Provider>
);
```

**Ce face `<VoteContext.Provider>`?**
- Înfășoară componentele copil (`children`)
- Pune datele în `value` prop
- Toate componentele copil pot accesa aceste date

**`value` prop:**
```javascript
value={{ votes, upVote, downVote }}

// E același lucru cu:
value={{
  votes: votes,
  upVote: upVote,
  downVote: downVote
}}
```

**Fluxul complet:**

```
1. VoteProvider se montează
   ↓
2. useState(0) creează votes = 0
   ↓
3. upVote și downVote sunt create
   ↓
4. value = { votes: 0, upVote, downVote }
   ↓
5. <VoteContext.Provider> primește value
   ↓
6. Toate componentele {children} au acces la value
```

### Pasul 3: Împachetăm componentele

```javascript
// main.jsx
import { VoteProvider } from './vote-provider';

createRoot(document.getElementById("root")).render(
  <VoteProvider>
    <App />
  </VoteProvider>
);
```

**Ce se întâmplă:**

```
1. React montează VoteProvider
   ↓
2. VoteProvider primește children = <App />
   ↓
3. VoteProvider creează state și funcții
   ↓
4. VoteProvider returnează:
   <VoteContext.Provider value={...}>
     <App />
   </VoteContext.Provider>
   ↓
5. App și toate componentele din interior au acces la value
```

**Cine apelează `VoteProvider`?**
- **React** o apelează automat când vede `<VoteProvider>`
- Tu doar o scrii în JSX, React o execută

### Pasul 4: Creăm custom hook (opțional dar recomandat)

```javascript
// use-context-votes.js
import { useContext } from "react";
import { VoteContext } from "./vote-context";

export function useContextVotes() {
  return useContext(VoteContext);
}
```

**De ce custom hook?**
- Mai simplu de folosit
- Mai puțin cod repetitiv
- Poți adăuga logică extra dacă vrei

**Cine apelează `useContext`?**
- **TU** o apelez prin custom hook: `useContextVotes()`
- `useContext` este apelat de **React** intern când hook-ul se execută

**Cum funcționează `useContext`?**

```
1. TU apelez: const data = useContextVotes();
   ↓
2. useContextVotes apelează: return useContext(VoteContext);
   ↓
3. useContext caută cel mai apropiat <VoteContext.Provider>
   ↓
4. Extrage value din Provider
   ↓
5. Returnează { votes, upVote, downVote }
```

### Pasul 5: Folosim datele în componente

```javascript
// Header.jsx
import { useContextVotes } from './use-context-votes';

export function Header() {
  const { votes } = useContextVotes();
  
  return <div>Total voturi: {votes}</div>;
}
```

```javascript
// ProductItem.jsx
import { useContextVotes } from './use-context-votes';

export function ProductItem() {
  const { votes, upVote, downVote } = useContextVotes();
  
  return (
    <div>
      <button onClick={upVote}>👍 Upvote</button>
      <span>{votes}</span>
      <button onClick={downVote}>👎 Downvote</button>
    </div>
  );
}
```

**Fluxul complet când user votează:**

```
1. Header și ProductItem montate
   ↓
2. Ambele apelez: useContextVotes()
   ↓
3. useContext(VoteContext) găsește Provider-ul
   ↓
4. Extrage: { votes: 0, upVote, downVote }
   ↓
5. Header afișează: "Total voturi: 0"
6. ProductItem afișează: "0"
   ↓
7. User dă click pe 👍 Upvote
   ↓
8. onClick={upVote} → TU (prin click) apelez upVote()
   ↓
9. upVote() în Provider:
   setVotes(0 + 1) → setVotes(1)
   ↓
10. State-ul votes se schimbă: 0 → 1
   ↓
11. Provider se re-renderizează cu value = { votes: 1, ... }
   ↓
12. TOATE componentele care folosesc useContextVotes se re-renderizează
   ↓
13. Header afișează: "Total voturi: 1"
14. ProductItem afișează: "1"
```

**Important:** Când state-ul în Context se schimbă, TOATE componentele care folosesc acel Context se re-renderizează automat!

## 💻 Codul esențial

### 1. Creăm Context

```javascript
// vote-context.js
import { createContext } from "react";

export const VoteContext = createContext();
```

**Ce face:**
- Importă funcția `createContext` din React
- Creează un context nou (cutia goală)
- Exportă contextul pentru a fi folosit

**Cine apelează:**
- **TU** apelez `createContext()` când fișierul se încarcă

### 2. Împachetăm în Provider

```javascript
// vote-provider.jsx
import { useState } from "react";
import { VoteContext } from "./vote-context";

export function VoteProvider({ children }) {
  const [votes, setVotes] = useState(0);

  const upVote = () => {
    setVotes(votes + 1);
  };

  const downVote = () => {
    setVotes(votes - 1);
  };

  return (
    <VoteContext.Provider value={{ votes, upVote, downVote }}>
      {children}
    </VoteContext.Provider>
  );
}
```

**Ce face:**
- Creează componenta Provider
- Gestionează state-ul (votes) și funcțiile (upVote, downVote)
- Înfășoară componentele copil în `<VoteContext.Provider>`
- Pune datele în prop-ul `value`

**Cine apelează:**
- **React** apelează `VoteProvider()` când vede `<VoteProvider>`
- **React** apelează `useState(0)` când componenta se montează
- **TU** apelez `upVote()` și `downVote()` la click

### 3. Custom Hook

```javascript
// use-context-votes.js
import { useContext } from "react";
import { VoteContext } from "./vote-context";

export function useContextVotes() {
  return useContext(VoteContext);
}
```

**Ce face:**
- Creează un custom hook pentru acces mai ușor
- Folosește `useContext` pentru a extrage datele din Context
- Returnează datele din Provider

**Cine apelează:**
- **TU** apelez `useContextVotes()` în componente
- **React** apelează `useContext(VoteContext)` intern

**De ce e util:**
```javascript
// Fără custom hook (mai lung):
import { useContext } from "react";
import { VoteContext } from "./vote-context";

const { votes, upVote } = useContext(VoteContext);

// Cu custom hook (mai scurt):
import { useContextVotes } from "./use-context-votes";

const { votes, upVote } = useContextVotes();
```

## 🎯 Utilizări practice

### 1. State Cart 🛒

**Problema:**
Header, Cart și Product pot accesa produsele din coș fără a trece props prin 5 componente!

```javascript
// CartContext.js
export const CartContext = createContext();

// CartProvider.jsx
export function CartProvider({ children }) {
  const [cart, setCart] = useState([]);

  const addToCart = (product) => {
    setCart([...cart, product]);
  };

  const removeFromCart = (productId) => {
    setCart(cart.filter(item => item.id !== productId));
  };

  return (
    <CartContext.Provider value={{ cart, addToCart, removeFromCart }}>
      {children}
    </CartContext.Provider>
  );
}

// useCart.js
export function useCart() {
  return useContext(CartContext);
}

// Header.jsx - Afișează numărul de produse
function Header() {
  const { cart } = useCart();
  return <div>Coș: {cart.length} produse</div>;
}

// Product.jsx - Adaugă în coș
function Product({ product }) {
  const { addToCart } = useCart();
  return <button onClick={() => addToCart(product)}>Adaugă în coș</button>;
}
```

**Fluxul:**

```
1. CartProvider înfășoară aplicația
   ↓
2. Header și Product folosesc useCart()
   ↓
3. User dă click pe "Adaugă în coș"
   ↓
4. onClick={() => addToCart(product)} → TU apelez addToCart()
   ↓
5. addToCart în Provider:
   setCart([...cart, product])
   ↓
6. State-ul cart se actualizează
   ↓
7. Header se re-renderizează automat
   ↓
8. Afișează numărul nou de produse
```

### 2. State Temă 🎨

**Problema:**
Home și alte componente schimbă tema aplicației instant

```javascript
// ThemeContext.js
export const ThemeContext = createContext();

// ThemeProvider.jsx
export function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light');

  const toggleTheme = () => {
    setTheme(theme === 'light' ? 'dark' : 'light');
  };

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}

// useTheme.js
export function useTheme() {
  return useContext(ThemeContext);
}

// App.jsx
function App() {
  const { theme } = useTheme();
  return <div className={theme === 'dark' ? 'bg-black' : 'bg-white'}>...</div>;
}

// ThemeSwitcher.jsx
function ThemeSwitcher() {
  const { theme, toggleTheme } = useTheme();
  return <button onClick={toggleTheme}>Temă: {theme}</button>;
}
```

**Fluxul:**

```
1. ThemeProvider înfășoară aplicația cu theme='light'
   ↓
2. App și ThemeSwitcher folosesc useTheme()
   ↓
3. App afișează fundal alb (bg-white)
   ↓
4. User dă click pe buton în ThemeSwitcher
   ↓
5. onClick={toggleTheme} → TU apelez toggleTheme()
   ↓
6. toggleTheme verifică: theme === 'light' ? setTheme('dark')
   ↓
7. State-ul theme devine 'dark'
   ↓
8. App se re-renderizează automat
   ↓
9. App afișează fundal negru (bg-black)
10. ThemeSwitcher afișează "Temă: dark"
```

### 3. Evită prop drilling ⚡

**Problema:**
Datele trec direct de la App la orice componentă, fără componente intermediare

**Înainte (Prop Drilling):**

```javascript
// App → Products → ProductList → ProductItem
// Trebuie să trimiți prin Products și ProductList chiar dacă nu le folosesc!

function App() {
  const [user, setUser] = useState(null);
  return <Products user={user} />;
}

function Products({ user }) {
  return <ProductList user={user} />;  // Nu folosește user
}

function ProductList({ user }) {
  return <ProductItem user={user} />;  // Nu folosește user
}

function ProductItem({ user }) {
  return <div>Bună, {user.name}!</div>;  // AICI folosește
}
```

**După (Context API):**

```javascript
// App (cu Provider) → ProductItem (folosește direct)
// Products și ProductList NU mai primesc props!

// UserProvider.jsx
function UserProvider({ children }) {
  const [user, setUser] = useState(null);
  return (
    <UserContext.Provider value={{ user, setUser }}>
      {children}
    </UserContext.Provider>
  );
}

// App.jsx
function App() {
  return (
    <UserProvider>
      <Products />  {/* Fără props! */}
    </UserProvider>
  );
}

function Products() {
  return <ProductList />;  {/* Fără props! */}
}

function ProductList() {
  return <ProductItem />;  {/* Fără props! */}
}

function ProductItem() {
  const { user } = useUser();  // Acces direct!
  return <div>Bună, {user.name}!</div>;
}
```

**Avantaje:**
- Products și ProductList nu mai primesc props inutile
- Dacă adaugi un nou prop în Context, nu modifici 3 componente
- Cod mai curat și ușor de întreținut

## 🎯 Recapitulare

### Ce este Context API?

**Soluție pentru a partaja date între componente fără prop drilling**

### Cei 3 pași

| Pas | Ce face | Cine apelează |
|-----|---------|---------------|
| **1. createContext()** | Creează containerul | **TU** - la import |
| **2. Provider** | Înfășoară componentele, pune datele | **React** - când vede `<Provider>` |
| **3. useContext()** | Extrage datele | **TU** - prin custom hook |

### Structura fișierelor

```
src/
  context/
    vote-context.js          ← Creează context
    vote-provider.jsx        ← Provider cu state
    use-context-votes.js     ← Custom hook
  components/
    Header.jsx               ← Folosește context
    ProductItem.jsx          ← Folosește context
  main.jsx                   ← Înfășoară în Provider
```

### Pattern complet

```javascript
// 1. Creează Context
export const VoteContext = createContext();

// 2. Creează Provider
export function VoteProvider({ children }) {
  const [votes, setVotes] = useState(0);
  const upVote = () => setVotes(votes + 1);
  
  return (
    <VoteContext.Provider value={{ votes, upVote }}>
      {children}
    </VoteContext.Provider>
  );
}

// 3. Custom Hook
export function useVotes() {
  return useContext(VoteContext);
}

// 4. Înfășoară în main.jsx
<VoteProvider>
  <App />
</VoteProvider>

// 5. Folosește în componente
const { votes, upVote } = useVotes();
```

### Când folosești Context?

✅ **DA - Folosește Context:**
- Date globale (user, theme, language)
- State partajat între multe componente
- Evită prop drilling (3+ nivele)

❌ **NU - Nu folosi Context:**
- State local (doar o componentă)
- State între 2 componente apropiate (folosește props)
- Date care se schimbă foarte des (performance!)
